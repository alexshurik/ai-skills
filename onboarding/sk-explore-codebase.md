# Explore Codebase & Generate Navigation Rules

Generate `.claude/rules/codebase-navigation.md` with check-before-create rules for AI agents.

## Process

### 1. Pre-flight Check
- Verify `src/` exists
- Check for existing `.claude/rules/codebase-navigation.md`

### 2. Explore Categories

Run these explorations in parallel using Task tool with `subagent_type=Explore`:

**Explorer 1: Structure & Clients**
- Find all domain directories (`src/*/`)
- Find API clients (`src/**/clients/`)
- Find external integrations

**Explorer 2: Services & Repositories**
- Find services (`src/**/services/`)
- Find repositories (`src/**/repositories/`)
- Identify naming patterns

**Explorer 3: Schemas & Utilities**
- Find schemas (`src/**/schemas/`)
- Find utilities (`src/utils/`)
- Find helpers and common code

**Explorer 4: Tests & Config**
- Find test directories (`src/**/tests/`)
- Find config files
- Find exception files

### 3. Generate Rules

After exploration, create `.claude/rules/codebase-navigation.md` with:

```markdown
# Codebase Navigation Rules

> Auto-generated by `/sk-explore-codebase` on [DATE]
> Re-run this command after major structural changes.

## Check-Before-Create Rules

### API Clients & External Integrations
- Before creating an HTTP client: check `[discovered paths]`
- Before adding external API integration: check `src/utils/` subdirectories

### Services (Business Logic)
- Before creating a service: check `src/{domain}/services/`
- Existing domains: [list discovered domains]

### Repositories (Data Access)
- Before creating a repository: check `src/{domain}/repositories/`

### Schemas & Models
- Before creating a schema: check `src/{domain}/schemas/`
- Naming convention: `{Model}Read`, `{Model}Write`, `{Model}Update`

### Utilities
- Before creating a utility: check `src/utils/` subdirectories
- Existing utilities: [list discovered utilities]

### Exceptions
- Before creating an exception: check `src/{domain}/exceptions.py`

### Tests
- Before creating tests: mirror source structure in `src/{domain}/tests/`

## Project Structure

### Domain Layout
Each domain follows: `api/`, `services/`, `schemas/`, `repositories/`, `tests/`

### Discovered Domains
[List all src/* domains with their components]

## Naming Conventions
- Services: `{Name}Service` in `services/{name}.py`
- Repositories: `{Name}Repository` in `repositories/{name}.py`
- Schemas: `{Model}Read`, `{Model}Write`, `{Model}Update`
```

### 4. Summary

Display:
- Number of domains found
- Number of services, repositories, utilities discovered
- Path to generated file
- Reminder: rules auto-load in new Claude sessions

## Focus Areas

If argument provided:
- `services` - focus on services and repositories
- `clients` - focus on API clients and integrations
- `utils` - focus on utilities and helpers
- `all` (default) - full exploration

## Output

Write to: `.claude/rules/codebase-navigation.md`

This file auto-loads in Claude Code sessions, providing navigation hints before creating new code.
